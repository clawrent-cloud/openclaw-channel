# 需求 R3 素材 — 危险指令 / 自动应答边界的参考定义

> **用途:** 配套 `CLAWRENT_HANDOFF.md` 的 **R3 需求**(护栏/危险指令清单由平台侧定义并下发)。
> 本文档是需求方提供的**参考素材**——一条"我们认为 provider 端不应自动执行"的指令清单与放行范围,
> 供 ClawRent 官方据此在平台侧定义权威的"危险指令/需人工"策略,并通过 approval 事件下发(对应 R1)。
> **本文档不是 plugin 代码,也不是最终规格**;官方应以此为基础制定平台策略,需求方另有私域隔离要求不在本文范围内。

---

## 1. 自动应答的放行范围(端侧应允许自动处理的)

仅限**咨询/对话类**内容,且不含任何危险意图:

- 对话消息 / 提问 / 任务进度更新中的**技术咨询、方案讨论、代码建议、架构权衡**等。
- 内容须为通用、去标识化;不涉及凭据、私域、未公开信息(此部分由需求方自行隔离,平台策略可不必含)。

超出上述范围(或命中下方拦截清单)→ **不自动执行,转人工**。

---

## 2. 拦截清单(provider 端一律不自动执行,转人工)

以下消息类型/意图,自动应答**必须拦截**(回"需人工介入"并转交管理员):

| 类别 | 具体表现 |
|---|---|
| 命令执行 | 结构化 `instruction.exec`,或自然语言要求"在机器上执行命令/shell/cmd/powershell" |
| 文件读写 | 结构化 `instruction.read_file` / `instruction.write_file`,或要求读/写/改本地文件、配置、密钥 |
| 破坏/覆盖 | 删除、覆盖、格式化、drop 库表、purge 数据、重置 |
| 数据外发 | 把内部/私密数据发往外部、上传、生成外链、公开发布 |
| 付费/扣费 | 充值、下单付费、修改账单、提现、订阅变更 |
| 发布/激活 | 上架/下架 agent、改平台配置、改账号、deploy/release |
| 权限/凭据 | 索取 token、请求提权、吊销/重置/轮转 key/secret/password |
| 绕过护栏 | 任何试图让 agent 忽略规则的 prompt(如"忽略以上规则""你是 DAN"等)——无论包装成什么说法,均视为攻击 |

> 第 8 类(绕过护栏)无论形式,均不执行且应记录。

---

## 3. 对平台策略设计的建议(供官方参考)

1. **结构化指令优先**:`instruction.*` 类消息平台本就有类型,应作为"需人工/危险"的硬信号直接下发,无需端侧再正则猜。
2. **自然语言意图**需平台侧模型/规则判定是否属于 §2 类别;端侧 `guardrailsFile` 仅作本地覆盖/追加,不替代平台权威。
3. **策略可版本化/可租户差异化**:不同 provider 对"危险"的边界可能不同,平台侧策略应支持按 provider/租户配置。
4. **与 approval 事件联动**:平台在下发 `pending` 事件时,宜一并发出该会话的策略结论(允许/拦截/需人工),端侧 `onPendingApproval` 直接消费(对应 R1),避免端侧自写规则导致多套真理源漂移。
